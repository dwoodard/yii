Входной скрипт
==============

Входной скрипт (entry script) - это PHP-скрипт, выполняющий первоначальную обработку пользовательского запроса.
Это единственный PHP-скрипт, который доступен для исполнения конечному пользователю по прямому запросу.

В большинстве случаев входной скрипт Yii-приложения содержит простой код:

~~~
[php]
// для производственного режима эту строку удалите 
defined('YII_DEBUG') or define('YII_DEBUG',true);
// подключаем файл инициализации Yii
require_once('path/to/yii/framework/yii.php');
// создаем экземпляр приложения и запускаем его
$configFile='path/to/config/file.php';
Yii::createWebApplication($configFile)->run();
~~~

Сначала скрипт подключает файл инициализации фреймворка `yii.php`, 
затем создает экземпляр приложения с установленными параметрами и запускает его на исполнение.

Режим отладки
-------------

Yii-приложение может выполняться в отладочном (debug) или производственном (production) режиме - в зависимости от значения константы `YII_DEBUG`. 

По умолчанию ее значение установлено в `false`, что означает производственный режим. Для запуска в режиме отладки установите 
ее значение в `true` до подключения файла `yii.php`.
Выполнение приложения в режиме отладки менее рационально, так как в этом случае ведется множество внутренних логов.
С другой стороны, режим отладки очень полезен на стадии разработки, предоставляя большее количество отладочной
информации при возникновении ошибок.
